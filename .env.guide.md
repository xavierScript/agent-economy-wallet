# Environment Variables Guide

This document explains the configuration variables used in the `.env` file for the Yanga Wallet project.

## NETWORK

- **`SOLANA_RPC_URL`**: The RPC endpoint for Solana network interactions. Default is `https://api.devnet.solana.com`.
- **`SOLANA_CLUSTER`**: The target Solana cluster. Valid options are `devnet`, `mainnet-beta`, or `testnet`.

## SECURITY

- **`WALLET_PASSPHRASE`**: Passphrase used to encrypt/decrypt all keypairs at rest. It must be at least 12 characters. The encryption uses AES-256-GCM, with a key derived via PBKDF2 (210,000 iterations, SHA-512).

## MASTER WALLET

Auto-funds newly created agent wallets from a designated funder keypair.

- **`MASTER_WALLET_SECRET_KEY`**: For quick testing, you can provide the base58 private key directly. Recommended only for development. For secure production use, follow the Secure Setup Protocol below to import the key without exposing it in the environment.
- **Secure Setup Protocol**:
  1. Paste your base58 private key into `MASTER_WALLET_SECRET_KEY`.
  2. Run `pnpm key:import` (Key is encrypted and saved to `~/.agent-economy-wallet/keys/`).
  3. Delete or comment out `MASTER_WALLET_SECRET_KEY` from your `.env` file.
  4. Set `MASTER_WALLET_KEY_LABEL` to the label chosen during import.
  5. Clear your shell history to ensure the key isn't stored in memory (e.g., `history -c && history -w` on bash).
- **`MASTER_WALLET_KEY_LABEL`**: Label assigned during the `pnpm key:import` process. Default is `master-funder`.
- **`AGENT_SEED_SOL`**: The amount of SOL to seed each new agent wallet. Default is `0.05` (approx. 100 simple transfers on devnet).
- **`OWNER_ADDRESS`**: The operator address used as a sweep destination when a wallet is closed via the TUI. Leave blank to skip the sweep.
- **`MERCHANT_RECEIVER_ADDRESS`**: The address that receives payments from the Merchant API paywall. Defaults to `OWNER_ADDRESS` if not set, but can be a different address if you want to separate merchant funds from the operator's main wallet.

## ON-CHAIN REGISTRY & REPUTATION

- **`REGISTRY_WALLET_ADDRESS`**: Public key of the registry coordination wallet (SPL Memo anchor). All merchants register by sending a transaction referencing this address. Buyer agents scan this address to discover active merchants.
- **`REPUTATION_PROGRAM_ID`**: The Solana program ID (base58) for the Anchor smart contract that tracks merchant reputation. If deployed locally/devnet, this should match your Anchor deploy coordinates.
- **`AGENT_NAME`**: The name of your merchant agent as it will appear in the `/.well-known/agent.json` service manifest.
- **`AGENT_DESCRIPTION`**: A short description of the services your merchant provides, also published in the service manifest.

## PROTOCOL REVENUE

- **`PROTOCOL_FEE_ADDRESS`**: The treasury wallet address (base58) that collects a percentage of every x402 payment made by buyer agents.
- **`PROTOCOL_FEE_BPS`**: The protocol fee structured in basis points (1 bp = 0.01%). Example: `50` means 0.5% of every transaction is automatically routed to the `PROTOCOL_FEE_ADDRESS`.


## KORA GASLESS RELAY

- **`KORA_RPC_URL`**: URL for the Kora gasless relay service. Default is `http://localhost:8080`.
- **`KORA_API_KEY`**: Optional API key (uncomment if `kora.toml` has `[kora.auth] api_key` set).
- **`KORA_SIGNER_PRIVATE_KEY`**: Path to the JSON keypair file that Kora uses as its fee-payer signer. Use a wallet you already own (e.g. Phantom) rather than generating a throwaway key — this keeps the signer address under your full control.
  - **Recommended setup** — export from Phantom once and convert to JSON:
    1. In Phantom: Settings → Security & Privacy → Export Private Key (copies a base58 string).
    2. Run the included helper script to convert it (no extra dependencies needed):
       ```bash
       node scripts/phantom-to-kora.mjs YOUR_BASE58_KEY
       ```
    3. Set `KORA_SIGNER_PRIVATE_KEY=kora/kora-signer.json` in `.env`. The raw base58 key never lives in the env file.
    4. Clear your terminal history: `history -c && history -w` (bash) or close and reopen the terminal.
  - The `kora/kora-signer.json` file is `.gitignore`d and will never be committed to source control.

## LOGGING

- **`LOG_LEVEL`**: Application log level. Valid options are `debug`, `info`, `warn`, or `error`. Default is `info`.
