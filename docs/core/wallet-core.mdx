# Wallet Core Architecture

The `wallet-core` package encapsulates all Solana-specific logic, cryptography, guardrails, and protocol-level integrations into a modular set of tightly controlled services. These services act as the secure foundation upon which the higher-level MCP logic operates.

## Sub-Components

### 1. Key Manager (`key-manager.ts`)

Handles secure AES-GCM keystore loading, creation and isolation.

- Encrypts plain-text seed phrases using standard PBKDF2 key derivations.
- Ensures the AI agent only ever sees references to public keys within the environment.

### 2. Policy Engine (`policy-engine.ts`)

The immutable guardrail for all asset transfers.

- Validates every outward bound transaction against human-enforced constraints.
- Enforces rolling 24-hour spending windows.
- Requires explicit `.env` whitelists for interactions (protects against interacting with completely unknown Solana program addresses).

### 3. Service Factory (`service-factory.ts`)

Simplifies instantiation. Automatically loads the appropriate environment variables. It wires up the RPC Connection, Audit Logger, Policy Engine, Master Funder, and Spl Token Services into one cohesive `WalletServices` context object passed to all tools.

### 4. Spl Token & Master Funder (`spl-token.ts` & `master-funder.ts`)

Handles complex programmatic interactions on chain. `master-funder.ts` enables "gasless" simulations, where a singular loaded devnet wallet pays for all transaction signatures across dynamically generated sub-agent wallets.

### 5. x402 Server & Client Protocols

A bespoke implementation for enforcing monetary headers over REST requests.

- **Server:** Decodes base64 signatures, verifies the token transferred went to the merchant address, matches the amount, and ensures the signature is not replayed by hashing recent signatures in an LRU cache.
- **Client:** The middleware that intercepts `402 Payment Required` errors and automatically triggers wallet payments.
